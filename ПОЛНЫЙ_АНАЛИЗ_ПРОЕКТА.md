## Основные скрипты

### 1. `zapret4rocket/z4r.sh` (757 строк)

**Главный скрипт установки и управления системой обхода блокировок.**

#### Переменные и настройки:
- `DEFAULT_VER="72.6"` - версия zapret по умолчанию
- Цветовые коды для терминала (plain, red, green, yellow, blue, pink, cyan и их жирные варианты)
- Определение пути скрипта и проверка наличия библиотек

#### Основные функции:

**`change_user()`** (строки 98-108)
- Определяет пользователя для запуска zapret (nobody или первый пользователь из /etc/passwd)
- Проверяет доступность через `--dry-run`
- Раскомментирует `WS_USER` в config.default

**`get_repo()`** (строки 111-141)
- Создает структуру директорий для стратегий
- Скачивает списки доменов из GitHub:
  - netrogat.txt (исключения)
  - russia-discord.txt, russia-youtube*.txt, tg_cidr.txt
- Загружает архив fake файлов (fake_files.tar.gz)
- Скачивает List.txt для каждой категории стратегий
- Создает пустые файлы стратегий (1.txt - 17.txt для TCP, 1.txt - 8.txt для UDP)
- Восстанавливает бэкапы из /opt/extra_strats и /opt/netrogat.txt
- Загружает config.default и скрипты для войсов (50-stun4all, 50-discord-media)
- Автоматически определяет nftables и меняет FWTYPE

**`remove_zapret()`** (строки 144-162)
- Останавливает zapret если запущен
- Выполняет uninstall_easy.sh если доступен
- Удаляет директории /opt/zapret и /tmp/zapret

**`version_select()`** (строки 165-214)
- Запрашивает версию zapret у пользователя
- Если Enter - определяет последнюю версию через GitHub API
- Использует 4 метода парсинга версии (sed -E, grep+cut, sed -r, awk)
- Валидирует формат версии (цифры и точки, максимум 4 символа)
- Fallback на DEFAULT_VER при ошибках

**`zapret_get()`** (строки 217-228)
- Определяет тип архива (VPS или OpenWRT embedded)
- Скачивает и распаковывает архив zapret
- Запускает install_bin.sh
- Удаляет ненужные бинарники (оставляет только для текущей архитектуры)
- Перемещает zapret в /opt/zapret

**`install_zapret_reboot()`** (строки 231-240)
- Запускает install_easy.sh
- Перезапускает zapret
- Проверяет запуск через pidof nfqws
- Вызывает check_access_list для проверки доступности

**`entware_fixes()`** (строки 243-285)
- **Для Keenetic:**
  - Загружает кастомный скрипт zapret для Entware
  - Устанавливает 000-zapret.sh в netfilter.d
  - Устанавливает S00fix для отключения checksum
  - Копирует 10-keenetic-udp-fix
- **Для MerlinWRT:**
  - Патчит install_easy.sh (отключает проверку USER)
  - Добавляет правило в firewall-start для автозапуска
- Общие действия:
  - Запускает install_bin.sh
  - Вызывает change_user()
  - Патчит linux_fw.sh и linux_iphelper.sh если нет sysctl
  - Отключает запрос на чтение readme в installer.sh
  - Создает симлинк в автозагрузку

**`get_panel()`** (строки 288-312)
- Запрашивает установку панели туннелирования для VPS
- Поддерживает: 3xui, marzban, wg, 3proxy
- Для 3proxy дополнительно устанавливает build-essential
- Загружает конфигурационные файлы для 3proxy

**`ttyd_webssh()`** (строки 315-405)
- Устанавливает web-терминал ttyd для доступа через браузер
- Запрашивает логин (можно отключить через "0")
- **Для VPS:**
  - Скачивает бинарник ttyd.x86_64
  - Создает systemd service на порту 17681
- **Для OpenWRT:**
  - Устанавливает через opkg/apk
  - Настраивает через UCI
- **Для Entware:**
  - Создает init.d скрипт S99ttyd
  - Проверяет прослушивание порта 17681
- Проверяет запуск через pidof

**`get_menu()`** (строки 408-591)
- **Главное интерактивное меню системы**
- Инициализирует провайдера, телеметрию, рекомендации
- Отображает статус стратегий и zapret
- Пункты меню:
  - **Enter** - переустановка/обновление zapret
  - **0** - выход
  - **01** - проверка доступности сервисов
  - **1** - подбор стратегий (вызывает strategies_submenu)
  - **2** - стоп/перезапуск zapret
  - **3** - показать autohostlist (недоступные домены)
  - **4** - удалить zapret
  - **5** - обновить конфиг и сбросить стратегии
  - **6** - добавить домен в исключения (netrogat.txt)
  - **7** - открыть config в nano
  - **8** - переключить скрипты bol-van / классические стратегии
  - **9** - переключить iptables/nftables
  - **10** - активировать UDP обход для игр (1026-65531)
  - **11** - управление FLOWOFFLOAD
  - **12** - безразборный режим TCP-443
  - **13** - установить web-терминал
  - **14** - управление провайдером
  - **777** - премиум функция
  - **999** - секретный пункт (только с премиум)

#### Логика выполнения (строки 593-757):
1. Определение ОС (os-release, entware_release)
2. Определение железа (keenetic, merlin, netcraze)
3. Определение типа системы (VPS, WRT, entware)
4. Проверка доступности GitHub API
5. Для VPS - запрос установки панели
6. Если zapret установлен - показ меню или быстрый подбор стратегий
7. Для entware - установка зависимостей
8. Создание /opt и переход в /tmp
9. Бэкап стратегий
10. Удаление старого zapret
11. Выбор версии
12. Запрос установки web-ssh
13. Скачивание zapret
14. Загрузка репозитория
15. Применение фиксов для entware/x-wrt
16. Установка и запуск

### 3. `lib/provider.sh` (119 строк)

**Определение провайдера и города для персонализации подсказок.**

**Переменные:**
- `PROVIDER_CACHE` - путь к кэшу провайдера
- `PROVIDER_MENU` - строка для отображения в меню
- `PROVIDER_INIT_DONE` - флаг инициализации

**`_detect_api_simple()`** (строки 8-73)
- Внутренняя функция определения провайдера
- Пробует ip-api.com (HTTP, формат line)
- Fallback на ipapi.co (HTTPS, отдельные запросы)
- Очищает результат (только латиница, цифры, пробелы, точки, дефисы, подчеркивания)
- Убирает дубликаты если провайдер = город
- Сохраняет в PROVIDER_CACHE

**`provider_init_once()`** (строки 75-91)
- Инициализация провайдера один раз за сессию
- Проверяет кэш, если пуст - вызывает _detect_api_simple()
- Загружает результат в PROVIDER_MENU

**`provider_force_redetect()`** (строки 93-103)
- Принудительное переопределение провайдера
- Удаляет кэш и вызывает _detect_api_simple()

**`provider_set_manual_menu()`** (строки 105-115)
- Ручной ввод провайдера и города
- Форматирует строку "Провайдер - Город"
- Сохраняет в кэш

### 4. `lib/telemetry.sh` (107 строк)

**Модуль телеметрии для сбора статистики использования стратегий.**

**Переменные:**
- `STATS_FORM_ID` - ID Google Forms
- `ENTRY_*` - поля формы (UUID, ISP, UDP, TCP, GV, RKN)
- `CACHE_DIR` - директория кэша
- `TELEMETRY_CFG` - конфиг телеметрии

**`init_telemetry()`** (строки 17-62)
- Инициализация телеметрии (спрашивает один раз)
- Загружает существующий конфиг
- Если статус не задан - запрашивает разрешение пользователя
- Генерирует UUID (из /proc/sys/kernel/random/uuid или md5 от времени)
- Сохраняет настройки в TELEMETRY_CFG

**`send_stats()`** (строки 65-105)
- Отправка статистики в Google Forms
- Проверяет разрешение пользователя
- Читает провайдера из provider.txt
- Определяет активные стратегии через get_active_strat_num()
- Отправляет POST запрос в фоне (тихий режим, таймаут 10 сек)
- Не блокирует выполнение скрипта

### 5. `lib/recommendations.sh` (64 строки)

**База подсказок по стратегиям на основе провайдера.**

**Переменные:**
- `RECS_URL` - URL файла рекомендаций
- `RECS_FILE` - локальный файл рекомендаций

**`update_recommendations()`** (строки 7-20)
- Обновление базы рекомендаций
- Проверяет возраст файла (если моложе 1 дня - пропускает)
- Скачивает recommendations.txt с GitHub
- Таймаут 5 секунд

**`show_hint()`** (строки 23-61)
- Показ подсказки по стратегиям для провайдера
- Принимает тип стратегии (UDP, TCP, GV, RKN)
- Читает провайдера из provider.txt
- Ищет строку в recommendations.txt (формат: ISP|UDP:X|TCP:Y|GV:Z|RKN:W)
- Парсит нужную часть через cut
- Выводит подсказку с цветовым оформлением

### 6. `lib/netcheck.sh` (60 строк)

**Проверка доступности ресурсов и получение доменов YouTube.**

**`get_yt_cluster_domain()`** (строки 3-32)
- Получение домена кластера YouTube для проверки видеопотока
- Делает запрос к redirector.xn--ngstr-lra8j.com (дважды для надежности)
- Конвертирует cluster_codename через маппинг символов
- Использует два массива символов для конвертации
- Возвращает домен вида "rr1---sn-XXXXX.googlevideo.com"
- Fallback на фиксированный домен при ошибке

**`check_access()`** (строки 34-48)
- Проверка доступности ресурса
- Проверяет TLS 1.2 (важно для ТВ и старых устройств)
- Проверяет TLS 1.3 (современные устройства)
- Таймаут 1 секунда для каждой проверки
- Выводит цветные сообщения о результате

**`check_access_list()`** (строки 50-59)
- Проверка списка популярных ресурсов:
  - youtube.com (YT TCP интерфейс)
  - googlevideo.com кластер (YT видеопоток)
  - meduza.io (RKN лист)
  - instagram.com (RKN лист)

### 8. `lib/strategies.sh` (206 строк)

**Логика работы со стратегиями обхода.**

**`get_active_strat_num()`** (строки 3-18)
- Определяет номер активной стратегии в папке
- Перебирает файлы от 1 до MAX
- Возвращает номер первого непустого файла
- Возвращает 0 если ничего не найдено

**`get_current_strategies_info()`** (строки 21-38)
- Генерирует строку статуса всех стратегий
- Проверяет UDP/YT, TCP/YT, TCP/GV, TCP/RKN
- Форматирует вывод: "YT_UDP:Def YT_TCP:2 YT_GV:0 RKN:5"
- Использует цветовое оформление (Def - серый, номер - зеленый)

**`try_strategies()`** (строки 41-130)
- Основная функция перебора стратегий
- Параметры:
  - `count` - количество стратегий
  - `base_path` - путь к папке стратегий
  - `list_file` - файл со списком доменов или /dev/null
  - `final_action` - дополнительное действие после сохранения
- Логика:
  1. Запрашивает номер стратегии для начала (или Enter)
  2. Очищает все файлы стратегий
  3. Цикл от выбранной стратегии до count:
     - Очищает предыдущую стратегию
     - Записывает текущую (копирует list_file или домен)
     - Для TCP стратегий - проверяет доступность:
       - googlevideo.com → получает кластер
       - пустой домен → youtube.com
       - кастомный → сам домен
     - Запрашивает подтверждение (1-сохранить, 0-отмена, Enter-далее)
     - При сохранении вызывает send_stats()
     - Выполняет final_action если задано
  4. Если цикл завершен без выбора - сбрасывает последнюю стратегию

**`Strats_Tryer()`** (строки 133-205)
- Главная функция подбора стратегий
- Принимает режим (1-4) или домен (строка)
- Режимы:
  - **"1"** - YouTube UDP QUIC (8 стратегий)
    - Показывает подсказку UDP
    - Использует List.txt из UDP/YT
  - **"2"** - YouTube TCP интерфейс (17 стратегий)
    - Показывает подсказку TCP
    - Использует List.txt из TCP/YT
  - **"3"** - googlevideo.com видеопоток (17 стратегий)
    - Убирает GV из YT листа
    - Устанавливает user_domain="googlevideo.com"
    - Показывает подсказку GV
    - Использует /dev/null (домен задан вручную)
  - **"4"** - RKN лист (17 стратегий)
    - Очищает все RKN стратегии
    - Устанавливает user_domain="meduza.io" для проверки
    - Показывает подсказку RKN
    - Использует List.txt из TCP/RKN
  - **"5"** - Кастомный домен (17 стратегий)
    - Запрашивает домен у пользователя
    - Использует /dev/null
    - Сохраняет домен в User/{N}.txt после выбора
    - Очищает temp/{N}.txt

### 9. `lib/submenus.sh` (210 строк)

**Подменю управления различными настройками.**

**`strategies_submenu()`** (строки 5-49)
- Подменю выбора режима подбора стратегий
- Показывает текущие стратегии
- Пункты:
  - 1 - YouTube UDP QUIC (8 вариантов)
  - 2 - YouTube TCP интерфейс (17 вариантов)
  - 3 - YouTube видеопоток/GV (17 вариантов)
  - 4 - RKN лист (17 вариантов)
  - 5 - Кастомный домен (17 вариантов)
  - 0/Enter - назад
- Вызывает Strats_Tryer() с соответствующим режимом
- Для пункта 5 запрашивает домен (поддерживает несколько через пробел)

**`flowoffload_submenu()`** (строки 51-105)
- Управление аппаратным ускорением
- Показывает текущее состояние FLOWOFFLOAD
- Варианты:
  - 1 - software (программное ускорение)
  - 2 - hardware (аппаратное NAT)
  - 3 - none (отключено)
  - 4 - donttouch (дефолт, не трогать)
- Меняет config, запускает install_prereq.sh и перезапускает zapret

**`tcp443_submenu()`** (строки 107-158)
- Управление безразборным режимом TCP-443
- Показывает текущую стратегию безразборного режима
- Запрашивает номер стратегии (1-17) или 0 для отключения
- При активации:
  - Очищает кастомные домены (User и temp)
  - Предлагает добавить .ru домены в исключения
  - Меняет строку в config (убирает "none.dom")
  - Перезапускает zapret
  - Проверяет доступность meduza.io
- При отключении:
  - Возвращает "none.dom" в config
  - Перезапускает zapret

**`provider_submenu()`** (строки 160-209)
- Управление провайдером и подсказками
- Показывает текущего провайдера
- Пункты:
  - 1 - указать провайдера вручную
  - 2 - определить заново (сбросить кэш)
  - 3 - обновить базу рекомендаций
  - 0/Enter - назад
- Вызывает соответствующие функции из provider.sh и recommendations.sh

### 10. `lib/actions.sh` (144 строки)

**Действия меню (бэкапы, обновления, переключатели).**

**`backup_strats()`** (строки 1-32)
- Резервное копирование стратегий и исключений
- Запрашивает подтверждение для каждого бэкапа
- Бэкап extra_strats → /opt/extra_strats
- Бэкап netrogat.txt → /opt/netrogat.txt
- При вводе "0" возвращает в меню

**`menu_action_update_config_reset()`** (строки 35-67)
- Обновление конфига со сбросом стратегий
- Показывает дату последнего обновления конфига
- Вызывает backup_strats()
- Останавливает zapret
- Удаляет lists, extra_strats, fake файлы
- Вызывает get_repo() для загрузки свежих файлов
- Вызывает change_user()
- Копирует config.default → config
- Запускает zapret

**`menu_action_toggle_bolvan_ports()`** (строки 69-102)
- Переключение между скриптами bol-van и классическими стратегиями
- **Режим скриптов bol-van:**
  - NFQWS_PORTS_UDP=443
  - Стратегия войса закомментирована (--skip)
  - Скрипты 50-stun4all и 50-discord-media установлены
- **Режим классических стратегий:**
  - NFQWS_PORTS_UDP=443,1400,3478-3481,5349,50000-50099,19294-19344
  - Стратегия войса раскомментирована
  - Скрипты удалены
- Перезапускает zapret после переключения

**`menu_action_toggle_fwtype()`** (строки 104-122)
- Переключение между iptables и nftables
- Меняет FWTYPE в config
- Запускает install_prereq.sh (переустанавливает зависимости)
- Перезапускает zapret

**`menu_action_toggle_udp_range()`** (строки 124-143)
- Активация/деактивация UDP обхода для игр (порты 1026-65531)
- **Включение:**
  - NFQWS_PORTS_UDP=1026-65531,443
  - Раскомментирует стратегию UDP игр
- **Выключение:**
  - NFQWS_PORTS_UDP=443
  - Закомментирует стратегию (--skip)
- Перезапускает zapret

### 16. `recommendations.txt` (534 строки)

**База рекомендаций по стратегиям для провайдеров.**

**Формат:** `Провайдер - Город|UDP:X,Y|TCP:X,Y|GV:X,Y|RKN:X,Y`

**Примеры:**
- `MTS - Moscow|UDP:0, 2, 3|TCP:1|GV:0, 1, 4|RKN:0`
