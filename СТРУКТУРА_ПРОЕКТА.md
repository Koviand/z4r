# Структура проекта z2r_mod

## Обзор проекта

Проект состоит из двух основных репозиториев:
1. **zapret4rocket** - основной репозиторий с системой обхода блокировок
2. **z4r** - репозиторий установщика и менеджера

---

## 📁 Структура директорий

### 1. Репозиторий `zapret4rocket/`

Основной репозиторий системы обхода блокировок на базе zapret.

#### 📄 Корневые файлы

- **`z4r.sh`** (757 строк) - Главный скрипт установки и управления системой
  - Определение ОС (VPS, OpenWRT, Keenetic, MerlinWRT)
  - Установка/обновление zapret
  - Управление стратегиями обхода
  - Интерактивное меню управления

- **`config.default`** (231 строка) - Конфигурационный файл с настройками стратегий
  - Настройки NFQWS (Netfilter Queue WebSocket)
  - Стратегии обхода для TCP/UDP
  - Настройки фильтрации по портам
  - Параметры для различных доменов (YouTube, Discord, Telegram и др.)

- **`README.md`** - Документация проекта с инструкциями по установке и использованию

- **`recommendations.txt`** - Рекомендации по стратегиям

- **`3proxy.cfg`** - Конфигурация прокси-сервера 3proxy

- **`del.proxyauth`** - Скрипт удаления аутентификации прокси

#### 📂 `lib/` - Модульные библиотеки (9 файлов)

Модульная архитектура для разделения функциональности:

1. **`ui.sh`** - UI-хелперы
   - `pause_enter()` - пауза с ожиданием Enter
   - `submenu_item()` - вывод пунктов подменю
   - `exit_to_menu()` - возврат в меню

2. **`provider.sh`** - Определение провайдера и города
   - `provider_init_once()` - однократная инициализация
   - `provider_force_redetect()` - принудительное переопределение
   - `provider_set_manual_menu()` - ручная установка через меню
   - Использует API: ip-api.com, ipapi.co

3. **`telemetry.sh`** - Телеметрия и статистика
   - `init_telemetry()` - инициализация телеметрии
   - `send_stats()` - отправка статистики в Google Forms

4. **`recommendations.sh`** - База подсказок по стратегиям
   - `update_recommendations()` - обновление базы подсказок
   - `show_hint()` - вывод подсказки по провайдеру

5. **`netcheck.sh`** - Проверка доступности ресурсов
   - `get_yt_cluster_domain()` - получение домена кластера YouTube
   - `check_access()` - проверка доступности ресурса
   - `check_access_list()` - проверка списка ресурсов

6. **`premium.sh`** - Премиум функции (пункты 777/999)
   - `rand_from_list()` - случайный выбор из списка
   - `spinner_for_seconds()` - спиннер на N секунд
   - `premium_get_or_set_title()` - получение/установка титула
   - `zefeer_premium_777()` - премиум функция 777
   - `zefeer_space_999()` - космическая функция 999

7. **`strategies.sh`** - Логика работы со стратегиями
   - `get_active_strat_num()` - определение активной стратегии
   - `get_current_strategies_info()` - информация о текущих стратегиях
   - `try_strategies()` - перебор стратегий
   - `Strats_Tryer()` - основной обработчик подбора стратегий

8. **`submenus.sh`** - Подменю управления
   - `strategies_submenu()` - подменю стратегий
   - `flowoffload_submenu()` - подменю flowoffload
   - `tcp443_submenu()` - подменю TCP443
   - `provider_submenu()` - подменю провайдера

9. **`actions.sh`** - Действия меню
   - `backup_strats()` - резервное копирование стратегий
   - `menu_action_update_config_reset()` - обновление конфига со сбросом
   - `menu_action_toggle_bolvan_ports()` - переключение портов bol-van
   - `menu_action_toggle_fwtype()` - переключение типа файрволла
   - `menu_action_toggle_udp_range()` - переключение UDP диапазона

#### 📂 `extra_strats/` - Дополнительные стратегии обхода

Структура стратегий для различных протоколов и доменов:

```
extra_strats/
├── TCP/
│   ├── GV/          # Google Video (googlevideo.com) - 17 стратегий (1-17.txt)
│   ├── RKN/         # РКН лист - 17 стратегий + List.txt
│   ├── User/        # Пользовательские домены - 17 стратегий
│   ├── YT/          # YouTube интерфейс - 17 стратегий + List.txt
│   └── temp/        # Временные стратегии - 17 файлов
└── UDP/
    └── YT/          # YouTube UDP QUIC - 8 стратегий + List.txt
```

**Принцип работы:**
- Файлы `1.txt`, `2.txt`, ... `17.txt` содержат домены для конкретной стратегии
- Если файл существует и не пуст - стратегия активна
- `List.txt` содержит основной список доменов для категории

#### 📂 `lists/` - Списки доменов и IP-адресов

- **`autohostlist.txt`** - автоматический список недоступных доменов
- **`cloudflare-ipset.txt`** - IP-адреса Cloudflare (IPv4)
- **`cloudflare-ipset_v6.txt`** - IP-адреса Cloudflare (IPv6)
- **`netrogat.txt`** - список исключений (домены, которые НЕ нужно обходить)
- **`russia-discord.txt`** - домены Discord для России
- **`russia-youtube-rtmps.txt`** - YouTube RTMPS трансляции
- **`russia-youtube.txt`** - YouTube домены для России
- **`russia-youtubeQ.txt`** - YouTube QUIC домены
- **`tg_cidr.txt`** - CIDR подсети Telegram

#### 📂 `fake/` - Бинарные файлы для обхода DPI

Файлы для имитации различных протоколов:

- **`http_fake_MS.bin`** - HTTP фейк для Microsoft
- **`quic_*.bin`** (7 файлов) - QUIC пакеты для обхода UDP блокировок
- **`syn_packet.bin`** - SYN пакет для TCP
- **`tls_clienthello_*.bin`** (18+ файлов) - TLS ClientHello пакеты:
  - Различные варианты для разных сайтов (google.com, vk.com, gosuslugi.ru и др.)
  - Специальные варианты (max_ru, 4pda_to, 2n, 6a)

#### 📂 `Entware/` - Скрипты для Entware (Keenetic/OpenWRT)

- **`000-zapret.sh`** - скрипт для netfilter.d (Keenetic)
- **`S00fix`** - скрипт автозапуска
- **`zapret`** - основной скрипт init.d для Entware

#### 📂 `.github/workflows/` - CI/CD

- **`update_recs.yml`** - автоматическое обновление рекомендаций

## 🔧 Архитектура системы

### Принцип работы

1. **Установка:**
   ```
   curl -O https://raw.githubusercontent.com/IndeecFOX/z4r/4/z4r && sh z4r
   ```
   - Скачивается скрипт `z4r`
   - Устанавливается в `/opt/bin/z4r` (или `/usr/local/bin/z4r`)
   - Загружаются библиотеки из `zapret4rocket/lib/`
   - Запускается `z4r.sh`

2. **Основной скрипт `z4r.sh`:**
   - Определяет ОС и оборудование
   - Устанавливает/обновляет zapret
   - Загружает стратегии и списки доменов
   - Предоставляет интерактивное меню управления

3. **Стратегии обхода:**
   - Стратегии хранятся в `extra_strats/`
   - Каждая стратегия - это набор доменов в файле `N.txt`
   - Конфигурация стратегий в `config.default`
   - Подбор стратегий через интерактивное меню

### Поддерживаемые платформы

- **VPS:** Ubuntu 22/24, Debian 12
- **OpenWRT/Keenetic:** Entware (Keenetic KN-3811)
- **MerlinWRT:** ASUS роутеры
- **Другие:** x-wrt, kwrt, istoreos

### Основные функции меню

1. Подбор стратегий для различных ресурсов
2. Управление исключениями (домены, которые не обходить)
3. Обновление конфигурации
4. Управление zapret (старт/стоп/перезапуск)
5. Настройка провайдера и города
6. Премиум функции (777/999)
7. Web-терминал для доступа через браузер
